# AI 논리 토론 시뮬레이터 개발 계획서

## 1. 프로젝트 개요

### 프로젝트명
AI 논리 토론 시뮬레이터

### 프로젝트 한 줄 설명
사용자가 특정 주제에 대해 자신의 입장을 입력하면, AI가 반대 입장에서 논리적으로 반박하고, 이후 전체 논리 구조와 약점을 분석하여 개선 전략까지 제시하는 웹 서비스.

### 프로젝트 목적
이 서비스의 목적은 단순한 텍스트 생성이 아니라, 사용자의 주장에 대해 반대 논리를 제시하고 논리적 약점을 분석함으로써 사고력, 설득력, 토론 대응력을 향상시키는 것이다.

### 핵심 차별점
일반적인 AI 생성 서비스와 달리 이 서비스는 다음 3단계를 가진다.

1. 사용자의 주장 생성 또는 입력
2. AI의 반대 논리 및 반박 시뮬레이션
3. 논리 구조 분석 및 개선 전략 제안

즉, 단순 생성기가 아니라 **시뮬레이션 + 분석 도구**로 동작해야 한다.

## 2. 타겟 사용자

### 1차 타겟
- 대학생
- 발표, 토론, 에세이, 논술, 면접을 준비하는 학생
- 자신의 주장 구조를 점검하고 싶은 사용자

### 2차 타겟
- 자기소개서/면접 답변의 논리성을 검토하고 싶은 취업 준비생
- 찬반 토론 연습이 필요한 학습자

---

## 3. 제품 목표

### 주요 목표
- 사용자가 주제와 입장을 입력하면 AI가 반대 논리를 생성할 수 있어야 한다.
- AI와의 토론 흐름이 단순 문답이 아니라 "구조화된 시뮬레이션"처럼 보여야 한다.
- 결과 화면에서 사용자의 주장에 대한 약점 분석과 개선 전략이 한눈에 보이도록 해야 한다.
- UI는 단순 채팅 앱이 아니라 SaaS 도구처럼 보여야 한다.

### 비목표
- 실시간 다중 사용자 토론 기능은 이번 MVP 범위에 포함하지 않는다.
- 음성 토론 기능은 포함하지 않는다.
- 사용자 간 공유 커뮤니티 기능은 포함하지 않는다.

---

## 4. 개발 범위(MVP)

### MVP에서 반드시 구현할 기능
1. 토론 주제 입력
2. 사용자 입장 선택 (찬성 / 반대)
3. 토론 스타일 선택 (선택사항)
   - 논리적
   - 공격적
   - 학술적
4. 사용자의 1차 주장 입력 또는 AI가 초안 생성
5. AI가 반대 입장에서 반박 생성
6. AI가 도전 질문 생성
7. 결과 분석 대시보드 제공
   - 논리 구조 분석
   - 약점 분석
   - AI 반박 요약
   - 개선 전략
8. 이전 시뮬레이션 기록 저장 및 조회

### 가능하면 구현하면 좋은 기능
1. 사용자 주장의 설득력 점수 표시
2. 논리 오류 유형 태깅
3. 재반박 생성 버튼
4. 결과를 카드/탭 UI로 구조화
5. 다국어 응답 설정

## 5. 사용자 시나리오

### 시나리오 A: 기본 사용 흐름
1. 사용자가 랜딩 페이지에 접속한다.
2. "토론 시작하기" 버튼을 누른다.
3. 토론 주제를 입력한다.
4. 자신의 입장을 선택한다.
5. 토론 스타일을 선택한다.
6. 자신의 주장 초안을 입력한다.
7. "AI 토론 시작" 버튼을 누른다.
8. AI가 반대 논리와 질문을 생성한다.
9. 사용자는 결과 대시보드에서 자신의 논리 구조와 약점을 확인한다.
10. 필요하면 개선 전략을 보고 자신의 주장을 수정한다.

### 시나리오 B: 기록 재확인
1. 사용자가 과거 시뮬레이션 기록 페이지에 들어간다.
2. 이전 주제를 클릭한다.
3. 당시 AI 반박과 분석 결과를 다시 확인한다.

## 8. 데이터 모델 초안

## 8.1 debates 테이블
토론 세션 기본 정보 저장

| 필드명 | 타입 | 설명 |
|---|---|---|
| id | bigint / int | PK |
| topic | varchar/text | 토론 주제 |
| user_stance | varchar | 사용자 입장 (찬성/반대) |
| debate_style | varchar | 토론 스타일 |
| user_argument | text | 사용자 주장 |
| output_language | varchar | 출력 언어 |
| created_at | datetime | 생성일시 |
| updated_at | datetime | 수정일시 |

## 8.2 debate_results 테이블
AI 응답 및 분석 결과 저장

| 필드명 | 타입 | 설명 |
|---|---|---|
| id | bigint / int | PK |
| debate_id | bigint / int | debates FK |
| ai_counter_argument | longtext | AI 반대 논리 |
| ai_challenging_questions | longtext | AI 질문 목록 |
| logic_analysis | longtext | 논리 구조 분석 |
| weakness_analysis | longtext | 약점 분석 |
| rebuttal_summary | longtext | AI 반박 요약 |
| improvement_strategy | longtext | 개선 전략 |
| created_at | datetime | 생성일시 |

### 참고
longtext 내부는 JSON 문자열 형태로 저장해도 된다.
예를 들어 질문 목록이나 분석 항목은 배열 JSON으로 보관 가능하다.

## 9. API / 서버 흐름 설계

## 9.1 기본 처리 흐름
1. 클라이언트가 토론 정보 입력
2. PHP 서버가 입력 검증
3. PHP 서버가 LLM 프롬프트 생성
4. 외부 AI API 호출
5. 응답 수신 후 구조화
6. DB 저장
7. 결과 반환
8. 프론트는 결과 대시보드 렌더링
